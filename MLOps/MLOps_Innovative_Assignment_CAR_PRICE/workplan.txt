1. Project Overview

This document provides a comprehensive, step-by-step workplan to build and deploy a Used Car Price Prediction application following the complete MLOps lifecycle. The project incorporates data engineering, machine learning model development, API serving, containerization, orchestration, monitoring, and automated retraining pipelines.

1.1 Objective
Build a production-grade ML application that:
Accepts user input (car attributes such as make, model, year, mileage, fuel type, etc.)
Processes the input through a trained ML model
Returns an accurate used car price prediction
Is scalable, monitored, and automatically retrained on new data

Step 2.1 — Missing Value Treatment
Identify columns with > 30% missing values — consider dropping them
Numerical columns: impute using median (robust to outliers)
Categorical columns: impute using mode or 'Unknown' placeholder
Use pandas .isnull().sum() to audit missing values per column

Step 2.2 — Outlier Detection and Removal
Apply IQR (Interquartile Range) method for numerical columns
Remove records where selling_price < 10,000 or > 10,000,000 INR
Remove records where km_driven > 500,000
Validate year: must be between 1990 and current year

Step 2.3 — Data Type Corrections
Parse torque column: extract numeric value from strings like '113Nm @ 1500rpm'
Strip units from mileage: convert '23.4 kmpl' to float 23.4
Strip units from engine: convert '1197 CC' to integer 1197
Strip units from max_power: convert '82 bhp' to float 82.0

Step 2.4 — Deduplication
Drop exact duplicate rows using df.drop_duplicates()
Flag near-duplicate entries with same name, year, and km_driven


Step 3.1 — Derive New Features
car_age: Calculate as current_year - year (e.g., 2025 - 2018 = 7)
brand: Extract the first word from name column (e.g., 'Maruti' from 'Maruti Swift')
model: Extract second word from name column
price_per_km: selling_price / km_driven (market efficiency metric)
is_first_owner: Binary flag if owner == 'First Owner'

Step 3.2 — Encoding Categorical Variables
Label Encoding: Apply to ordinal columns (owner, seller_type)
One-Hot Encoding: Apply to nominal columns (fuel, transmission)
Target Encoding: Apply to high-cardinality column (brand, model)

Step 3.3 — Feature Scaling
Apply StandardScaler to: km_driven, engine, max_power, mileage
Save the scaler as scaler.pkl for use during inference

Step 3.4 — Feature Selection
Use correlation matrix to remove highly correlated features (threshold > 0.90)
Apply SHAP values post-training to validate feature importance
Final selected features: car_age, km_driven, fuel, transmission, engine, max_power, seats, brand, seller_type, owner

Step 12.1 — Retraining Triggers
Scheduled: Airflow DAG runs weekly retraining every Sunday at 2 AM
Drift-based: Trigger if prediction drift score > 0.15 (monitored via Grafana)
Manual: GitHub Actions workflow_dispatch for on-demand retraining

Step 12.2 — Retraining Workflow
Download latest data from source API or data lake
Append to existing dataset and validate quality
Retrain model with same pipeline (feature eng + training code)
Compare new model metrics against production model in MLflow
If new model improves RMSE by > 2%: promote to production
Update Docker image tag and trigger Kubernetes rollout
Monitor post-deployment for 24 hours before full rollout
